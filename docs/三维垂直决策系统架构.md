# 三维垂直决策系统架构设计

## 1. 核心定位

**不做：** 通用人生决策预测系统（冷启动问题、预测不准、无法验证）

**要做：** 三维垂直决策过程管理系统

- 职业选择决策
- 人际关系决策  
- 升学规划决策

**核心价值：** 不是预测未来，而是让决策过程更系统、更有依据、更可追踪

## 3. 统一决策流程（三个维度通用）

### 3.1 决策建模阶段（30分钟）

**目标：** 把模糊的想法变成结构化的决策模型

**统一流程：**
```
1. 问题定义
   - 你要决策什么？
   - 为什么现在要决策？
   - 不决策会怎样？

2. 选项列举
   - 有哪些可能的选择？
   - 每个选项的核心是什么？
   - 还有没想到的选项吗？

3. 关键假设识别
   - 每个选项基于什么假设？
   - 哪些假设是关键的？
   - 哪些假设可以验证？

4. 约束条件
   - 时间约束
   - 资源约束
   - 关系约束
   - 能力约束

5. 决策目标
   - 短期目标（1-6个月）
   - 中期目标（6-18个月）
   - 长期目标（18个月+）
```

**输出：** 一份结构化的决策文档

### 3.2 信息收集阶段（1-2周）

**目标：** 收集真实数据，验证关键假设

**统一方法：**
```
1. 识别信息缺口
   - 哪些信息是必需的？
   - 哪些信息是可选的？
   - 哪些信息可以获取？

2. 生成收集清单
   - 需要了解什么？
   - 从哪里获取？
   - 谁可以提供？

3. 对接真实资源
   - 职业：岗位JD、薪资数据、行业报告
   - 人际：关系现状、沟通记录、影响分析
   - 升学：学校信息、录取数据、就业去向

4. 记录收集结果
   - 收集到什么信息？
   - 信息可信度如何？
   - 还缺什么信息？
```

**输出：** 信息收集报告

### 3.3 方案分析阶段（1-2天）

**目标：** 基于真实数据分析各选项

**统一维度：**
```
1. 可行性分析
   - 能力匹配度
   - 资源充足度
   - 时间合理性
   - 风险可控性

2. 收益分析
   - 短期收益
   - 长期收益
   - 隐性收益
   - 机会成本

3. 风险分析
   - 失败概率
   - 失败代价
   - 可逆程度
   - 应对方案

4. 关系影响分析
   - 谁支持？为什么？
   - 谁反对？为什么？
   - 影响权重如何？
   - 如何沟通？
```

**输出：** 方案对比矩阵

### 3.4 决策执行阶段（持续）

**目标：** 把决策变成可执行的行动计划

**统一结构：**
```
1. 行动计划
   - 第一步做什么？
   - 时间节点
   - 检查点
   - 调整机制

2. 风险预案
   - 可能出什么问题？
   - 如何识别？
   - 如何应对？
   - 退出机制

3. 进度追踪
   - 每周记录
   - 关键指标
   - 偏差分析
   - 及时调整

4. 定期复盘
   - 1个月复盘
   - 3个月复盘
   - 6个月复盘
```

**输出：** 执行追踪报告

## 4. 三个垂直维度的差异化设计

### 4.1 职业选择决策引擎

**特点：** 有大量真实数据可对接

**核心功能：**
```
1. 岗位匹配分析
   - 对接招聘网站API（拉勾、Boss直聘、LinkedIn）
   - 分析目标岗位的真实要求
   - 计算能力匹配度

2. 薪资预期分析
   - 真实薪资数据
   - 行业薪资曲线
   - 涨薪路径

3. 学习路径规划
   - 技能缺口分析
   - 学习资源推荐
   - 时间成本估算

4. 转型案例库
   - 相似背景的成功案例
   - 失败案例分析
   - 关键经验提取

5. 行业趋势分析
   - 行业发展趋势
   - 岗位需求变化
   - 技术栈演进
```

**知识图谱结构：**
```
节点类型：
- 岗位节点（JD、薪资、要求）
- 技能节点（技能树、学习资源）
- 公司节点（规模、文化、发展）
- 行业节点（趋势、机会、风险）
- 用户能力节点（当前技能、经验）

关系类型：
- 岗位-要求-技能
- 技能-前置-技能
- 岗位-属于-行业
- 用户-掌握-技能
- 用户-目标-岗位
```

### 4.2 人际关系决策引擎

**特点：** 高度个性化，依赖用户输入

**核心功能：**
```
1. 关系网络建模
   - 关键人物识别
   - 关系类型标注
   - 影响权重计算

2. 关系动态追踪
   - 关系状态变化
   - 沟通记录
   - 情感温度

3. 冲突分析
   - 利益冲突点
   - 期待差异
   - 沟通障碍

4. 沟通策略建议
   - 沟通目标
   - 沟通方式
   - 话术建议
   - 时机选择

5. 关系维护提醒
   - 重要日期
   - 定期联系
   - 关系投资
```

**知识图谱结构：**
```
节点类型：
- 人物节点（姓名、关系、特征）
- 事件节点（互动记录、关键事件）
- 情感节点（情感状态、变化趋势）
- 期待节点（对方期待、自己期待）
- 冲突节点（冲突点、严重程度）

关系类型：
- 人物-关系-人物（亲属、朋友、同事）
- 人物-影响-决策（支持、反对、中立）
- 人物-参与-事件
- 事件-导致-情感变化
- 冲突-涉及-人物
```

### 4.3 升学规划决策引擎

**特点：** 有明确的数据和标准

**核心功能：**
```
1. 学校信息库
   - 学校排名
   - 专业实力
   - 录取数据
   - 就业去向

2. 录取概率分析
   - 历年分数线
   - 录取比例
   - 竞争态势
   - 个人匹配度

3. 备考规划
   - 时间规划
   - 科目安排
   - 资源推荐
   - 进度追踪

4. 专业选择分析
   - 专业前景
   - 课程设置
   - 就业方向
   - 个人兴趣匹配

5. 成本收益分析
   - 学费成本
   - 时间成本
   - 机会成本
   - 预期收益
```

**知识图谱结构：**
```
节点类型：
- 学校节点（排名、地域、特色）
- 专业节点（课程、方向、就业）
- 考试节点（科目、难度、分数线）
- 用户状态节点（当前水平、目标）
- 就业节点（行业、岗位、薪资）

关系类型：
- 学校-开设-专业
- 专业-要求-考试
- 专业-通向-就业
- 用户-目标-学校
- 用户-当前-水平
```

## 5. 知识图谱的三视图设计

### 5.1 视图切换逻辑

```
用户进入决策：
1. AI识别决策类型（职业/人际/升学）
2. 自动切换到对应视图
3. 展示相关子图
4. 支持手动切换视图
```

### 5.2 视图间的关联

```
职业决策 ←→ 人际关系
- 职业选择会影响人际关系
- 人际关系会影响职业机会

职业决策 ←→ 升学规划
- 升学是为了更好的职业发展
- 职业目标决定升学方向

人际关系 ←→ 升学规划
- 家人意见影响升学决策
- 升学选择影响人际圈子
```

### 5.3 统一的图谱操作

```
所有视图支持：
1. 节点搜索
2. 关系筛选
3. 子图导出
4. 时间轴查看
5. 影响路径分析
```

### 6.2 数据库设计

```sql
-- 决策记录表（通用）
CREATE TABLE decisions (
    id VARCHAR(50) PRIMARY KEY,
    user_id VARCHAR(50),
    decision_type ENUM('career', 'relationship', 'education'),
    title VARCHAR(200),
    description TEXT,
    status ENUM('modeling', 'collecting', 'analyzing', 'executing', 'completed'),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 决策选项表（通用）
CREATE TABLE decision_options (
    id VARCHAR(50) PRIMARY KEY,
    decision_id VARCHAR(50),
    title VARCHAR(200),
    description TEXT,
    feasibility_score FLOAT,
    risk_score FLOAT,
    benefit_score FLOAT,
    final_score FLOAT
);

-- 信息收集表（通用）
CREATE TABLE information_collection (
    id VARCHAR(50) PRIMARY KEY,
    decision_id VARCHAR(50),
    question TEXT,
    answer TEXT,
    source VARCHAR(100),
    credibility FLOAT,
    collected_at TIMESTAMP
);

-- 执行追踪表（通用）
CREATE TABLE execution_tracking (
    id VARCHAR(50) PRIMARY KEY,
    decision_id VARCHAR(50),
    action_item TEXT,
    due_date DATE,
    status ENUM('pending', 'in_progress', 'completed', 'blocked'),
    notes TEXT,
    updated_at TIMESTAMP
);

-- 职业资源表
CREATE TABLE career_resources (
    id VARCHAR(50) PRIMARY KEY,
    job_title VARCHAR(200),
    company VARCHAR(200),
    salary_range VARCHAR(100),
    requirements TEXT,
    source VARCHAR(100),
    updated_at TIMESTAMP
);

-- 关系网络表
CREATE TABLE relationships (
    id VARCHAR(50) PRIMARY KEY,
    user_id VARCHAR(50),
    person_name VARCHAR(100),
    relationship_type VARCHAR(50),
    influence_weight FLOAT,
    emotional_closeness FLOAT,
    last_contact DATE,
    notes TEXT
);

-- 升学资源表
CREATE TABLE education_resources (
    id VARCHAR(50) PRIMARY KEY,
    school_name VARCHAR(200),
    major VARCHAR(200),
    admission_score INT,
    tuition DECIMAL(10,2),
    employment_rate FLOAT,
    source VARCHAR(100),
    updated_at TIMESTAMP
);
```
