## 当前核心目标

**让 LoRA 微调真正模仿用户，使决策副本能做出有个性化差异的推演。**

实现路径：游戏冷启动 + 长期对话提取 → 融合训练数据 → LoRA 微调 → 决策推演体现个性化差异。

## 任务二：个性化对比演示准备

### 演示方案

**对比一：同一用户，微调前 vs 微调后**

同一个问题（如"要不要辞职创业"），分别用：
- 基座模型推演 → 截图/录屏
- 该用户 LoRA 微调后推演 → 截图/录屏

对比点：事件描述是否更具体、是否体现该用户的决策风格。

**对比二：不同用户，同一问题**

两个用户（一个游戏中选保守、一个选激进），对同一问题分别推演，展示输出差异。

**答辩时的说法：**
> "左边是基座模型的推演，第一个月是'了解相关信息'这样的泛泛描述。右边是经过用户 A 的对话数据微调后的推演，第一个月是'联系前同事张某了解目标公司内部情况，同时评估现有工作的离职成本'——它知道用户认识谁，也知道用户倾向于谨慎决策。"

### 准备步骤

- [ ] 构造用户 A（保守型）：玩游戏时选稳定选项，对话中多表达"稳妥"、"评估风险"
- [ ] 构造用户 B（激进型）：玩游戏时选高风险选项，对话中多表达"机会"、"快速行动"
- [ ] 各积累 20+ 条对话后触发训练
- [ ] 对同一问题分别推演，截图保存

## 任务三：决策推演鲁棒性（按需处理）

你认为一般情况下没有问题，但建议在准备演示前跑 10 次推演，确认 fallback 触发率在可接受范围内。

如果发现 fallback 率偏高，优先调整 `num_events`（从 8 减到 4-5），输出更短，解析成功率更高。
