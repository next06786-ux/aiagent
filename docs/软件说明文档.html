一、系统概述

1.1 产品定位

择境 ChoiceRealm 是一款基于人工智能的智能决策推演系统，通过多Agent协同、知识图谱推理和平行宇宙模拟技术，帮助用户在职业发展、教育升学、人际关系等重大决策场景中做出更明智的选择。

核心理念

通过AI模拟不同选择的未来发展路径，让用户在做出决策前就能"看到"各种可能的结果，从而降低决策风险，提升决策质量。

多Agent协同推演

5个专业AI Agent（技能发展、职业人脉、财务规划、心理状态、市场环境）并行分析，模拟不同选择的未来12个月发展路径。

个性化知识图谱

构建用户的个人知识图谱，包含技能树、人脉网络、学习路径、职业发展等多维度信息，支持复杂关系推理。

对话式信息采集

通过自然语言对话深入了解决策背景，智能提取关键信息，降低用户输入负担。

实时可视化

WebSocket实时推送AI思考过程，3D图谱展示决策路径，让AI推理过程透明可见。

三大垂直领域

专注职业发展、教育升学、人际关系三大决策场景，提供专业化的决策支持。

智能洞察引擎

自动分析用户行为模式，发现潜在机会和风险，提供主动式决策建议。

1.2 核心价值

  • 降低决策风险：通过AI模拟提前预见不同选择的可能结果
  • 节省时间成本：自动化信息收集和分析，快速生成决策建议
  • 提升决策质量：多维度量化评估，避免主观偏见
  • 个性化推荐：基于用户画像和历史数据，提供定制化方案
  • 知识沉淀：构建个人知识图谱，积累决策经验
  • 透明可解释：实时展示AI推理过程，让决策依据清晰可见

1.3 应用场景

职业决策

跳槽、转行、晋升、创业等职业发展决策，评估不同选择的成功概率、薪资变化、技能匹配度等。

教育升学

考研、留学、专业选择、院校选择等教育决策，分析录取概率、学习路径、就业前景等。

人际关系

人脉拓展、关系维护、团队协作等社交决策，评估关系强度、网络价值、协作效果等。

2.3 数据流

典型决策推演流程

  1. 用户输入: 用户描述决策问题（如"我应该跳槽吗？"）
  2. 信息采集: AI通过对话深入了解背景（当前状况、目标、顾虑）
  3. 知识提取: 从知识图谱提取用户画像、技能树、人脉网络
  4. 选项生成: 基于算法生成3个可行方案
  5. 多Agent推演: 5个Agent并行模拟12个月时间线
  6. 实时推送: WebSocket推送每月Agent状态到前端
  7. 结果评估: 量化评分、风险识别、机会发现
  8. 推荐输出: 排序推荐最优方案，提供行动建议

三、核心功能

3.1 决策推演系统

系统的核心功能，通过多Agent协同模拟不同决策选项的未来发展路径。

5个专业Agent分工

  • 技能发展Agent: 评估技能匹配度、学习曲线、能力提升路径，计算技能差距和学习进度
  • 职业人脉Agent: 分析人脉资源、社交网络、内推机会，评估人脉质量和行业连接
  • 财务规划Agent: 计算薪资变化、生活成本、投资回报，监控财务跑道和现金流
  • 心理状态Agent: 评估压力水平、自我效能、适应能力，监测动力和韧性变化
  • 市场环境Agent: 监控行业趋势、岗位需求、竞争态势，分析市场波动和机会窗口

阶段1: 信息采集

通过对话式AI深入了解决策背景，自动提取关键信息（当前状况、目标、顾虑、约束条件），降低用户输入负担。

阶段2: 选项生成

基于知识图谱和决策算法，自动生成3个可行方案，用户可自定义修改和调整。

阶段3: 并行推演

5个Agent并行模拟12个月时间线，每个Agent独立评估并在关键节点协同决策。

阶段4: 结果呈现

3D决策图谱可视化，量化评估（成功概率、风险指数、预期收益），实时展示AI思考过程。

3.2 知识图谱系统

构建用户的个人知识图谱，支持复杂关系推理和路径规划。

用户画像

技能、经历、教育背景、性格特征、价值观等多维度信息，形成完整的个人档案。

技能树

技能层级关系、掌握程度、学习路径、技能依赖关系，支持技能缺口分析。

人脉网络

社交关系、影响力评估、推荐路径、关系强度，支持人脉价值分析。

职业图谱

岗位关系、转型路径、市场需求、薪资范围，支持职业规划和机会发现。

教育图谱

院校关系、专业路径、录取要求、就业前景，支持升学决策和路径规划。

决策历史

历史决策记录、结果反馈、经验沉淀，支持决策模式分析和持续优化。

知识图谱查询示例

// 查询用户的技能树和可转型岗位
MATCH (u:User {user_id: $user_id})-[:HAS_SKILL]->(s:Skill)
MATCH (s)-[:REQUIRED_BY]->(j:Job)
WHERE NOT (u)-[:WORKED_AS]->(j)
RETURN j.title, COUNT(s) as skill_match_count
ORDER BY skill_match_count DESC
LIMIT 5

// 查询用户的人脉网络中的推荐机会
MATCH (u:User {user_id: $user_id})-[:KNOWS*1..2]-(contact:User)
MATCH (contact)-[:WORKS_AT]->(company:Company)
MATCH (company)-[:HAS_OPENING]->(job:Job)
RETURN job, company, contact, LENGTH(path) as distance
ORDER BY distance ASC

5.2 多Agent协同机制

5个专业Agent独立演化并在关键节点协同决策，模拟真实的决策过程。

Agent协同流程

  1. 独立演化: 每个Agent基于自己的专业领域独立评估
  2. 状态共享: Agent之间共享当前状态，相互影响
  3. 交互检测: 识别Agent间的协同效应和冲突
  4. 决策点投票: 在关键决策点，所有Agent投票决定
  5. 综合评估: 汇总所有Agent的评估结果
# Agent协同示例
async def simulate_month(self, month, context):
    # 第一轮：独立演化
    agents_state = {}
    for agent_name, agent in self.agents.items():
        state = await agent.evolve(month, context, agents_state)
        agents_state[agent_name] = state
    
    # 第二轮：检测交互
    interactions = self._detect_interactions(month, agents_state)
    
    # 第三轮：识别决策点
    decision_points = self._identify_decision_points(month, agents_state)
    
    return {
        'month': month,
        'agents_state': agents_state,
        'interactions': interactions,
        'decision_points': decision_points
    }

7.3 知识图谱接口

获取知识图谱

GET /api/v5/knowledge-graph/{user_id}

// 响应
{
  "nodes": [
    {
      "id": "node_1",
      "type": "skill",
      "name": "Python",
      "level": 8
    }
  ],
  "edges": [
    {
      "source": "node_1",
      "target": "node_2",
      "type": "REQUIRES"
    }
  ]
}

添加知识节点

POST /api/v5/knowledge-graph/node

// 请求体
{
  "user_id": "user001",
  "name": "Python",
  "type": "skill",
  "category": "programming",
  "attributes": {
    "level": 8,
    "years": 3
  }
}
