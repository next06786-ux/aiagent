# LifeSwarm 模型压缩优化融合方案

## 总体目标

将 **llmquant** 和 **OBR** 两个先进的模型压缩技术融入 LifeSwarm，通过 **离线压缩 + 在线推理** 的策略，在保证模型性能的前提下：

- 降低基座模型显存占用：**20GB → 3-5GB**
- 压缩用户 LoRA 权重：**~50MB → ~12MB/用户**
- 加速推理速度：**~40% 的时延提升**

## 架构设计

```
LifeSwarm 推理架构（Transformers 原生）
│
├─ 基座模型 (Qwen3.5-9B)
│  ├─ 原始方案：FP16 → 20GB 显存
│  ├─ 压缩方案：W4A4KV4 + 50% 稀疏（OBR）→ 3-5GB 显存
│  └─ 支持动态加载 LoRA adapter
│
├─ 用户 LoRA 权重
│  ├─ 训练完成后自动量化（llmquant 4-bit）
│  ├─ 量化后：~12MB/用户
│  └─ 支持与量化基座模型融合推理
│
└─ 质量监控
   ├─ 量化前后性能评估
   ├─ 激活统计与能量分析
   └─ 自适应 rank 分配建议
```

## 核心融合方案

### 1. 基座模型压缩（OBR 主导）

#### 目标
用 OBR 的 FlatQuant 方案对 Qwen3.5-9B 执行一次性离线压缩。

#### 关键指标
- **量化策略**：W4A4KV4（权重、激活、KV cache 均为 4 bit）
- **稀疏度**：50% 非结构化稀疏
- **压缩比**：~4-5x（显存），~2-3x（计算量）
- **精度损失**：<0.5 perplexity 增长（在 WikiText2）

#### 实施路径

```python
# 1. 离线压缩脚本（一次性运行）
external_repos/OBR/FlatQuant/main.py
  ├─ 加载 Qwen3.5-9B
  ├─ 执行 W4A4KV4 量化 + 50% 稀疏
  └─ 导出压缩模型（支持 GPTQ/bitsandbytes 格式）

# 2. 集成到项目
backend/model_compression/
  ├─ compress_base_model.py        # OBR 压缩包装脚本
  ├─ load_quantized_model.py       # 量化模型加载器
  └─ eval_quantized_model.py       # 压缩后质量评估
```

#### 集成代码框架

```python
# backend/model_compression/compress_base_model.py
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer
from external_repos.OBR.FlatQuant import main as obr_compress

def compress_base_model_with_obr(
    model_name: str = "Qwen/Qwen3.5-9B",
    output_dir: str = "models/qwen3.5-9b-w4a4kv4-sparse50",
    sparsity_ratio: float = 0.5,
    bits: int = 4
):
    """
    使用 OBR 压缩基座模型
    
    Args:
        model_name: HuggingFace 模型名称
        output_dir: 压缩模型输出目录
        sparsity_ratio: 稀疏度（0.5 = 50%）
        bits: 量化位数（4 为 W4A4）
    
    Returns:
        压缩后的模型路径
    """
    # 调用 OBR 的 FlatQuant 压缩
    config = {
        "model": model_name,
        "w_bits": bits,
        "a_bits": bits,
        "k_bits": bits,
        "v_bits": bits,
        "k_asym": True,
        "v_asym": True,
        "k_groupsize": 128,
        "v_groupsize": 128,
        "sparsity_ratio": sparsity_ratio,
        "output_dir": output_dir,
        "ppl_eval": True  # 评估 perplexity
    }
    
    # 执行压缩
    obr_compress.run_compression(config)
    
    return output_dir
```

#### 加载量化模型

```python
# backend/model_compression/load_quantized_model.py
from transformers import AutoModelForCausalLM, BitsAndBytesConfig

def load_quantized_model_for_inference(
    model_path: str,
    quantization_type: str = "int4"  # 或 "int8"
):
    """
    加载量化后的模型用于推理
    """
    if quantization_type == "int4":
        bnb_config = BitsAndBytesConfig(
            load_in_4bit=True,
            bnb_4bit_use_double_quant=True,
            bnb_4bit_quant_type="nf4",
            bnb_4bit_compute_dtype=torch.bfloat16
        )
    else:
        bnb_config = BitsAndBytesConfig(load_in_8bit=True)
    
    model = AutoModelForCausalLM.from_pretrained(
        model_path,
        quantization_config=bnb_config,
        device_map="auto",
        trust_remote_code=True
    )
    
    return model
```

### 2. 用户 LoRA 权重压缩（llmquant 主导）

#### 目标
训练完成后，自动对 LoRA 权重进行 4-bit 量化。

#### 关键指标
- **量化位数**：4-bit per-channel
- **压缩比**：4x（存储空间）
- **推理损失**：<0.1% 精度下降

#### 实施路径

```python
# 修改 backend/lora/auto_lora_trainer.py
class AutoLoRATrainer:
    def __init__(self, user_id: str):
        self.user_id = user_id
        self.quantizer = None  # 将在初始化时设置
        
    async def train_lora(self):
        """训练 LoRA，完成后自动量化"""
        # ... 现有训练逻辑 ...
        
        # 训练完成后执行量化
        await self._quantize_lora_weights()
    
    async def _quantize_lora_weights(self):
        """使用 llmquant 量化 LoRA 权重"""
        from external_repos.llmquant.binary import quantize_to_4bit_perchannel
        
        lora_path = f"models/lora/{self.user_id}"
        adapter_config = load_adapter_config(lora_path)
        
        # 量化 lora_A 和 lora_B
        for layer_name in adapter_config["target_modules"]:
            lora_a = torch.load(f"{lora_path}/{layer_name}_a.pt")
            lora_b = torch.load(f"{lora_path}/{layer_name}_b.pt")
            
            # 4-bit 量化（per-channel）
            lora_a_q, _, _ = quantize_to_4bit_perchannel(lora_a)
            lora_b_q, _, _ = quantize_to_4bit_perchannel(lora_b)
            
            # 保存量化权重
            torch.save(lora_a_q, f"{lora_path}/{layer_name}_a_q4.pt")
            torch.save(lora_b_q, f"{lora_path}/{layer_name}_b_q4.pt")
        
        # 标记为已量化
        adapter_config["quantized"] = True
        adapter_config["quantization_type"] = "int4"
        save_adapter_config(lora_path, adapter_config)
```

#### 推理集成

```python
# backend/llm/llm_service.py 中修改加载逻辑
def load_lora_adapter_for_inference(
    model,
    user_id: str,
    use_quantized: bool = True
):
    """
    加载用户 LoRA adapter
    支持自动选择量化或非量化版本
    """
    from peft import PeftModel
    
    lora_path = f"models/lora/{user_id}"
    adapter_config = load_adapter_config(lora_path)
    
    # 选择量化或非量化版本
    if use_quantized and adapter_config.get("quantized"):
        # 加载量化版本
        lora_path_use = f"{lora_path}_q4"
    else:
        lora_path_use = lora_path
    
    # 使用 PeftModel 加载
    model = PeftModel.from_pretrained(
        model,
        lora_path_use,
        is_trainable=False
    )
    
    return model
```

### 3. 质量监控与自适应优化（llmquant 分析能力）

#### 目标
监控量化后的模型质量，支持自适应 rank 分配。

#### 核心功能

```python
# backend/model_compression/quality_monitor.py
from external_repos.llmquant.residual_analyzer import ResidualAnalyzer

class ModelCompressionMonitor:
    """量化模型质量监控"""
    
    def __init__(self, base_model_path: str):
        self.base_model_path = base_model_path
        self.analyzer = ResidualAnalyzer()
    
    def evaluate_quantization_quality(
        self,
        model_fp16,
        model_quantized,
        eval_dataset,
        metrics: List[str] = ["perplexity", "residual_energy"]
    ):
        """
        评估量化后的模型质量
        
        Returns:
            {
                "perplexity_increase": 0.3,  # WikiText2 增长
                "residual_energy": {...},    # 逐层残差能量
                "layer_quality": {...},      # 逐层量化保真度
                "recommendation": "可部署"
            }
        """
        results = {}
        
        # 1. 评估 Perplexity
        if "perplexity" in metrics:
            ppl_fp16 = self._eval_perplexity(model_fp16, eval_dataset)
            ppl_q = self._eval_perplexity(model_quantized, eval_dataset)
            results["perplexity_fp16"] = ppl_fp16
            results["perplexity_quantized"] = ppl_q
            results["perplexity_increase"] = ppl_q - ppl_fp16
        
        # 2. 逐层残差能量分析
        if "residual_energy" in metrics:
            energy_stats = self.analyzer.analyze_residual_distribution(
                model_fp16, model_quantized
            )
            results["residual_energy"] = energy_stats
        
        # 3. 生成部署建议
        if results.get("perplexity_increase", 0) < 0.5:
            results["recommendation"] = "✓ 可直接部署"
        elif results.get("perplexity_increase", 0) < 1.0:
            results["recommendation"] = "⚠ 可部署，建议监控推理质量"
        else:
            results["recommendation"] = "✗ 不建议部署，考虑调整量化参数"
        
        return results
    
    def suggest_lora_rank_allocation(
        self,
        model,
        eval_dataset,
        target_budget: int = 64  # 默认 rank=64
    ):
        """
        基于模型特性建议 LoRA rank 分配
        
        使用 llmquant 的激活熵统计和稳定秩计算
        """
        from external_repos.llmquant.run import allocate_ranks
        
        # 收集激活统计
        activation_entropies = self._collect_activation_entropies(model, eval_dataset)
        
        # 使用 llmquant 的贪心分配算法
        rank_allocation = allocate_ranks(
            layer_sensitivities={},  # 可选：基于 Hessian
            total_budget=target_budget * model.config.num_hidden_layers,
            use_activation_entropy=True,
            layer_activation_entropies=activation_entropies
        )
        
        return rank_allocation
```

## 性能预期

| 指标 | 原始 | 压缩后 | 提升 |
|------|------|--------|------|
| 显存占用 | 20GB | 3-5GB | 4-5x ↓ |
| 推理延迟 | 100ms | 60-70ms | 30-40% ↓ |
| 精度 (PPL) | 基线 | +0.3-0.5 | <1% ↓ |
| LoRA 存储 | 50MB | 12MB | 4x ↓ |

## 兼容性检查

- ✅ Transformers >= 4.35
- ✅ bitsandbytes >= 0.40（4-bit 量化）
- ✅ PEFT >= 0.4（LoRA 管理）
- ✅ Qwen3.5-9B 支持

## 风险与缓解

| 风险 | 描述 | 缓解方案 |
|------|------|---------|
| 精度损失 | 量化可能导致精度下降 | 离线验证，设置阈值 |
| 兼容性 | 量化模型与某些 LoRA 插件不兼容 | 在 PEFT 最新版本上测试 |
| 显存溢出 | 推理过程中偶发 OOM | 使用梯度检查点 |
| 重新训练需求 | 量化模型需要 calibration | 自动 calibration 数据集 |

## 后续优化方向

1. **动态量化**：根据运行时负载调整量化位数
2. **混合精度**：关键层保持 FP16，其余层量化
3. **KV Cache 优化**：使用 llmquant 的激活缩放优化 KV Cache
4. **分布式推理**：多 GPU 推理加速
