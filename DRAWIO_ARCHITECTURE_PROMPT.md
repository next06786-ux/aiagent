## 系统概述

### 项目定位
LifeSwarm 是一个基于 AI 的智能生活助手系统，提供多人格决策推演、知识图谱管理、智慧洞察等功能。

### 技术栈
- **前端**: React (Web) + HarmonyOS (移动端)
- **后端**: Python FastAPI
- **数据库**: MySQL + Redis + Neo4j + FAISS
- **AI**: Qwen/DeepSeek LLM + RAG 系统
- **部署**: Docker Compose

### 核心特性
1. **7人格决策副本系统** - 多视角决策推演
2. **三维知识星图** - 人际关系/教育升学/职业发展
3. **智慧洞察引擎** - 实时 Agent + 三层混合架构
4. **平行人生游戏** - 塔罗牌决策逻辑收集
5. **社交系统** - 好友管理 + 树洞世界
6. **智能日程** - 自动生成与推荐

### 数据库使用场景


### 5. 用户认证与会话管理流程

```
┌─────────────────────────────────────────────────────────────┐
│                    认证与会话管理                             │
└─────────────────────────────────────────────────────────────┘

注册流程
┌──────────────────────────────────────────────────────────┐
│  POST /api/auth/register                                  │
│         ↓                                                 │
│  验证:                                                     │
│  • 用户名格式 (3-20字符，字母数字下划线)                   │
│  • 邮箱格式                                                │
│  • 密码长度 (≥6字符)                                       │
│  • 用户名/邮箱唯一性                                        │
│         ↓                                                 │
│  创建用户:                                                 │
│  • 密码加密 (bcrypt)                                       │
│  • 生成 user_id                                            │
│  • 存储到 MySQL users 表                                   │
│         ↓                                                 │
│  生成 JWT Token:                                           │
│  • payload: {user_id, username, email}                    │
│  • 过期时间: 7天                                           │
│         ↓                                                 │
│  返回:                                                     │
│  • token                                                  │
│  • 用户信息                                                │
└──────────────────────────────────────────────────────────┘
         ↓
登录流程
┌──────────────────────────────────────────────────────────┐
│  POST /api/auth/login                                     │
│         ↓                                                 │
│  验证:                                                     │
│  • 用户名/邮箱存在性                                        │
│  • 密码正确性 (bcrypt.verify)                              │
│         ↓                                                 │
│  生成 JWT Token                                            │
│         ↓                                                 │
│  更新登录状态:                                             │
│  • Redis: user:login:{token} = user_id (7天)              │
│  • MySQL: users.last_login = now()                        │
│         ↓                                                 │
│  异步加载用户系统:                                          │
│  • 知识图谱 (InformationKnowledgeGraph)                    │
│  • RAG 系统 (ProductionRAGSystem)                          │
│         ↓                                                 │
│  返回:                                                     │
│  • token                                                  │
│  • 用户信息                                                │
└──────────────────────────────────────────────────────────┘
         ↓
请求认证
┌──────────────────────────────────────────────────────────┐
│  每个 API 请求携带 Authorization Header                    │
│         ↓                                                 │
│  Authorization: Bearer {token}                            │
│         ↓                                                 │
│  验证流程:                                                 │
│  1. 解析 JWT Token                                         │
│  2. 验证签名和过期时间                                      │
│  3. 从 Redis 检查登录状态                                  │
│  4. 提取 user_id                                           │
│         ↓                                                 │
│  如果验证失败:                                             │
│  • 返回 401 Unauthorized                                  │
│  • 前端跳转到登录页                                        │
│         ↓                                                 │
│  如果验证成功:                                             │
│  • 继续处理请求                                            │
│  • user_id 可用于业务逻辑                                  │
└──────────────────────────────────────────────────────────┘
         ↓
登出流程
┌──────────────────────────────────────────────────────────┐
│  POST /api/auth/logout                                    │
│         ↓                                                 │
│  清理:                                                     │
│  • Redis: 删除 user:login:{token}                         │
│  • 前端: 清除本地存储的 token                              │
│         ↓                                                 │
│  可选: 清理用户系统缓存                                     │
│  • 释放知识图谱连接                                        │
│  • 释放 RAG 系统资源                                       │
└──────────────────────────────────────────────────────────┘

启动时自动登录检查
┌──────────────────────────────────────────────────────────┐
│  startup_login_checker.py                                 │
│         ↓                                                 │
│  检查 Redis 中的活跃会话                                   │
│         ↓                                                 │
│  对每个活跃用户:                                           │
│  • 预加载知识图谱                                          │
│  • 预加载 RAG 系统                                         │
│         ↓                                                 │
│  优化启动体验:                                             │
│  • 用户登录后系统已就绪                                    │
│  • 减少首次请求延迟                                        │
└──────────────────────────────────────────────────────────┘
```
