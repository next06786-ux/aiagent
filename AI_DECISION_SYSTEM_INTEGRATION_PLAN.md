## 项目定位

**LifeSim AI - 基于多模态AI的个人生活决策与风险规避系统**

核心理念: 通过AI模拟"平行宇宙",帮助用户在重大决策前看到不同选择的未来,规避风险,做出最优决策。

## 详细集成方案

### 第1步: 心理测评系统 (新增)

**目标:** 构建用户性格基座

**集成点:** 用户注册后首次登录

**新建文件:**

- `backend/personality/personality_test.py` (心理测评逻辑)
- `backend/personality/questions.json` (20道测评题)
- `harmonyos/entry/src/main/ets/pages/PersonalityTest.ets` (前端测评页面)

**后端API:**
```
POST /api/personality/test
GET  /api/personality/profile/{user_id}
```

**测评维度:**
1. 大五人格 (开放性/尽责性/外向性/宜人性/神经质)
2. 决策风格 (理性分析型 vs 直觉感性型)
3. 风险偏好 (规避/中性/追求)
4. 生活优先级 (健康/事业/家庭/社交/财富/自由)
5. 价值观 (成就导向 vs 关系导向)

**数据存储:**
- 存入 `database/models.py` 的 `UserProfile` 表
- 作为后续所有决策的"基座"

---

### 第2步: LoRA自动化模型微调 (已有系统)

**目标:** 为每个用户训练专属的个性化AI模型

**已有文件:**
- `backend/lora/auto_lora_trainer.py` - 自动训练器
- `backend/lora/lora_model_manager.py` - 模型管理器
- `backend/decision/lora_decision_analyzer.py` - 决策分析器

**核心功能:**
```python
LoRA个性化 = 
    用户性格基座 (来自测评) +
    RAG对话数据 (自动收集) +
    决策反馈数据 (闭环学习) +
    自动微调训练 (每周或触发)
```

**训练策略:**
- 自动收集：从RAG系统获取用户对话
- 智能触发：每周定时 或 累积5个决策反馈
- 增量训练：基于上一版本继续训练
- 版本管理：v1, v2, v3... 自动保存

**集成方式:**
- 决策模拟时使用LoRA模型生成个性化推荐
- 决策反馈自动生成训练数据
- 后台定时任务自动训练

---

### 第3步: 动态画像更新 (新增)

**目标:** 用户会变,模型也要变

**集成点:** 后台定时任务

**新建文件:**
- `backend/personality/dynamic_profile_updater.py`

**触发条件:**
1. 时间触发: 每周日凌晨3点
2. 数据触发: 累积100条新对话
3. 事件触发: 重大决策后

**更新内容:**
- 短期状态 (每天): 情绪/压力/睡眠
- 中期变化 (每月): 习惯/社交模式
- 长期演化 (每季度): 性格漂移/价值观转变

**数据来源:**
- RAG记忆系统 (对话分析)
- 传感器数据 (行为模式)
- 知识图谱 (关系变化)

---

### 第4步: 平行宇宙模拟器 (基于已有digital_twin)

**目标:** 模拟多个决策路径的未来

**集成点:** 增强 `backend/digital_twin/digital_twin.py`

**新建文件:**
- `backend/decision/parallel_universe_simulator.py`

**核心功能:**
```python
输入: 决策问题 + 3-5个选项
输出: 每个选项的未来时间线 (12个月)

for each 选项:
    使用 digital_twin.simulate_step() 模拟12个月
    记录每个月的: 健康/财富/幸福/压力/关系/成就
```

**集成方式:**
调用你已有的 `DigitalTwin` 类,
为每个决策选项创建独立的模拟实例

---

### 第5步: 风险评估引擎 (新增)

**目标:** 量化每个决策的风险

**新建文件:**
- `backend/decision/risk_assessment_engine.py`

**6大风险维度:**

1. 健康风险: 压力过大/睡眠不足/心理问题
2. 财务风险: 收入中断/债务增加/储蓄耗尽
3. 社交风险: 孤独/关系破裂/社交隔离
4. 时间风险: 机会成本/年龄压力/时间浪费
5. 情绪风险: 焦虑/抑郁/后悔/挫败感
6. 学习风险: 失败概率/能力不足/竞争压力

**风险等级:**
- 低风险 (0-3分): 绿色
- 中风险 (4-6分): 黄色
- 高风险 (7-8分): 橙色
- 极高风险 (9-10分): 红色

**风险缓解建议:**
每个高风险项自动生成缓解方案

---

### 第6步: 游戏化可视化 (前端新增)

**目标:** 像玩游戏一样看到未来

**新建文件:**
- `harmonyos/entry/src/main/ets/pages/ParallelUniverseMap.ets`
- `harmonyos/entry/src/main/ets/components/TimelineAnimation.ets`

**核心组件:**

   - 颜色编码: 绿(好)/黄(中)/红(差)

2. **风险雷达图**
   - 6边形雷达图
   - 6个顶点代表6大风险
   - 面积越大风险越高

3. **时间线动画**
   - 播放按钮,看到12个月的变化
   - 关键事件标注
   - 状态曲线图

4. **决策建议卡片**
   - 推荐方案 (AI选择)
   - 成功概率
   - 主要风险
   - 缓解方案
   - 最坏情况
   - 后悔指数

### 第7步: 决策验证系统 (增强已有decision_tracker)

**目标:** 验证AI预测准确性,持续优化

**集成点:** 增强 `backend/digital_twin/decision_tracker.py`

**新增功能:**

1. **自动追踪**
   - 3个月后自动提醒用户反馈
   - 对比预测 vs 实际结果
   - 计算准确率

2. **学习优化**
   - 预测准确 → 增强该模式权重
   - 预测失败 → 调整模型参数
   - 反馈到强化学习系统

3. **统计报告**
   - 总决策数
   - 准确率
   - 用户满意度
   - 风险规避成功率

## 技术实现细节

### RTX 3050 + LoRA微调方案

**已实现配置:**
```python
# 基础模型: Qwen-0.8B (轻量级)
model_name = "Qwen/Qwen3.5-0.8B"
# 显存需求: ~2GB (推理) + ~4GB (训练)
# 推理速度: ~20 tokens/s
# 训练时间: ~3-5分钟/20条数据

# LoRA配置 (轻量版)
lora_config = {
    "r": 4,                    # LoRA秩
    "lora_alpha": 16,          # 缩放因子
    "target_modules": ["q_proj", "v_proj"],  # 只训练注意力层
    "lora_dropout": 0.1
}

# 训练配置
training_config = {
    "min_data_size": 20,       # 最少20条对话
    "train_interval_days": 7,  # 每7天训练一次
    "num_epochs": 1,           # 训练1轮
    "batch_size": 1            # 批次大小1（显存安全）
}
```

**混合方案 (已实现):**
- 日常对话: 使用Qwen API (快速)
- 决策分析: 使用用户LoRA模型 (个性化)
- LoRA训练: 本地RTX 3050 (隐私保护)
